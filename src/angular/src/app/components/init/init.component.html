TESTE TÉCNICO

Desafio Técnico para Desenvolvedor Full Stack (Java e Angular) - Nível Pleno

Objetivo:

Desenvolver uma aplicação web que facilite o processo de recrutamento interno para os colaboradores da empresa. A aplicação deve permitir aos usuários pesquisar e candidatar-se a vagas internas, passando por filtros de requisitos específicos.

Requisitos Funcionais:

  • Autenticação e Autorização: Implementar um sistema de autenticação seguro para os usuários.
    • Não deu tempo - ❌
  • Cadastro de Vagas: Permitir que os administradores cadastrem novas vagas, incluindo informações como título, descrição e requisitos. Possibilitar a atualização e exclusão de vagas.
      • Crud básico
    1. Savar - ✅
    2. Editar - ✅
    3. Remover - ✅
    4. Torná-la responsiva - ✅
  • Candidatura a Vagas: Implementar um sistema de candidatura às vagas, onde os usuários podem expressar interesse em uma posição específica. Enviar notificações aos responsáveis pela vaga e ao candidato.
    1. Mandar via SMS o interesse - ❌
    2. Mandar via e-mail o interesse - ❌
  • Painel do Candidato (Bônus): Criar um painel para os candidatos acompanharem o status de suas candidaturas e receberem feedbacks.
    • Não deu tempo - ❌
  • Avaliação de Candidatos (Bônus): Se possível, implementar um sistema de avaliação de candidatos pelos responsáveis pela vaga, incluindo filtros de requisitos ou tempo de empresa.
    • Não deu tempo - ❌

Diferenciais:

  • Adaptar a página para dispositivos móveis (torná-la responsiva). - ✅
  • Liberação da aplicação utilizando Docker. - ✅
  • Utilização de boas práticas de UX na solução. - ✅
  • Boa documentação de código e de serviços. - ✅
  • Testes do código. - ❌

Escopo:

  • Desenvolver os módulos de frontend e backend de forma separada.
  • O desenvolvimento do backend deve ser feito em Java 8+.
  • O desenvolvimento do frontend deve ser feito em Angular.
  • Preferencialmente utilizar Spring Boot 2.0+ com toda sua stack para o desenvolvimento do backend.
  • É aceitável utilizar algumas respostas estáticas em determinadas porções da aplicação.
  • Não é necessário submeter uma aplicação que cumpra cada um dos requisitos descritos, mas o que for submetido deve funcionar.
  • Utilizar um banco de dados relacional PostgreSQL.

O que será avaliado:

  • O código será avaliado seguindo os seguintes critérios: manutenabilidade, clareza e limpeza de código; resultado funcional; entre outros fatores.
  • Não esqueça de documentar o processo necessário para rodar a aplicação.
